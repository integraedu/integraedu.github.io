Linguagens e plataformas
O aplicativo (parte mobile), será desenvolvido para
as plataformas Android e IOS, será usado o
framework criado pela equipe de desenvolvimento
do Facebook o React Native, o mesmo consegue
gerar o aplicativo para duas plataformas em
somente um código, com este propósito de manter o
desenvolvimento unificado para as duas plataformas
será utilizado o desenvolvimento híbrido de
aplicativos, porém em sua versão web, usará o
ReactJs, que e especifico para criar Sistemas Web,
no modo Onepage ,todos os dados organizados e
consumidos, sendo administrado pelo framework
referido.
O desenvolvimento híbrido, usam tecnologias como
(HiperText Markup Language v5), CSS3 (Cascading
Style Sheets) e Javascript. A vantagem de utilizar o
desenvolvimento híbrido é que não precisa ser
desenvolvido separadamente o aplicativo em cada
plataforma, evitando assim o trabalho duplo o que
transforma o desenvolvimento final da aplicação
uma redução significativa no orçamento e tempo.
Como no desenvolvimento de uma aplicação
complexa a definição de que a mesma será
desenvolvida para versão mobile e web, foi
debatido e resolveu-se que a melhor solução seria
criar um serviço web, em que o mesmo aceitara
requisições ou chamadas das duas plataformas web
e mobile e assim ao ser solicitados as requisições,
retornará as respostas, a cada perfil da aplicação e
tipo de requisição (Get, Post, Delete, Put, Trace).
Assim com o desenvolvimento definido neste modelo
há inúmeras vantagens entre elas, custo e tempo,
comunidade expressiva, e consegue ajudar e retirar
duvidas, outra vantagem e a forma de comunicação,
que se dará pelo protocolo HTTP (Protocolo de
transferência de hipertexto), e o retorno e envio será
no formato JSON (Notação de Objetos JavaScript),
sendo um formato leve e que trafega com rapidez e
robustez, somente disponibilizando um servidor e
acesso à internet.
A criação do servidor de comunicação com a
camada web e mobile, se dará, com a criação de um
serviço Rest (Transferência de Estado
Representacional), que uma abstração da
arquitetura World Wide Web (Web), e mesma será
implementada em linguagem de programação Java
e usará o SpringBoot em sua versão 2.0, e usará
Framework Spring em sua versão 5.
Contudo, ainda é necessário armazenar os dados,
foi escolhido um banco de dados o Firebase Realtime
Database onde os dados são armazenados como
objetos JSON, o banco de dados como uma árvore
JSON hospedada na nuvem (cloud), o Firebase e um
banco de dados NOSQL e foi comprado pelo
Google em outubro de 2014, e o mesmo está em
grande crescimento.
